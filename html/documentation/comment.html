Commenting

Introduction:

I was recently going through my 2 year old Java program when it hit me. I did not understand my own code at a glance. But why not? I wrote the code myself. The variable names looked weird, the methods and their parmeters looked weird. There were no comments, and if they were present they were badly written.

Funny thing is, at the time it was a group project and no one complained. Because we were all present in one room whenever we worked on it we didn’t feel the need to comment through it. BUT this is bad practice. The present me is suffering because the old me didn’t take the time to comment though the source.

This tutorial serves to anwser the questions

  • what are comments?
  • how to comment.
  • how not to comment.

What then are comment?

Commenting as an art is the use of high level language, English (whichever language you speak), to describe what your program does.

We shall examin some example code, and study how comments should be used in them.

Case A:

                    
        // Let x be length
        // Let y be width
        // Let A be area of square
                        
        int x;
        int y;
        int a;
                        
        a = x * y;

Without knowing what the question was, the code written makes some sense. This is heavily due to the presence of comments. If we metally ommit them the variables x, y, a will not make sense, or at least it will take the reader a long time to figure out what the code means.

With the comments serving as and explanations you are able to deduce that the code calculates the Area of a Rectangle.
