Makefiles with an example

This tutorial in no way scratches the surface of a makefile and what is can do. Rather, it serves as a guide to understanding the pusrpose of a mailefile, the building blocks of a simple make file, and how to write one efficiently.

Think of this as a starter kit for students who will be using makefiles in the future and need an introductory course.

In order for us to slay this beast of a topic, let us consider the following example

add.h:
        
    // add.h file
    int add(int x, int y);
add.cpp
        
    // add.cpp
    #include "add.h"

    int add(int x, int y)
    {
        return x + y;
    }
main.cpp:
        
    // main.cpp
    #include <iostream>
    #include "add.h" // Insert contents of add.h at this point. Note use of double quotes here.

    int main()
    {
        std::cout << "The sum of 3 and 4 is " << add(3, 4) << '\n' ;
        return 0;
    }

Next open your terminal.

cd into directory of these files.

type:

    
    Blaq TuTu $ g++ -o hello main.cpp add.cpp add.h -I.

insert gif for running code with g++ here

This compiles the two .cpp files and names the executable hello. The -I. is included so that g++ will look in the current directory (.) for the include file hellomake.h. Without a makefile, the typical approach to the test/modify/debug cycle is to use the up arrow in a terminal to go back to your last compile command so you don't have to type it each time, especially once you've added a few more .c files to the mix.

Unfortunately, this approach to compilation has two downfalls. First, if you lose the compile command or switch computers you have to retype it from scratch, which is inefficient at best. Second, if you are only making changes to one .c file, recompiling all of them every time is also time-consuming and inefficient. So, it's time to see what we can do with a makefile.
