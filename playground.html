The idea of a CLI gpes back to the mid 20th Century. It was one one of the only ways that people had to communicate with the computer.

Today, terminal usually means a piece of software but it originally meant a physical piece of hardware for interacting with the computer. Before terminals, there was the use of punch cards to insert information into computers loading a batch all at once.

The terminal was a major improvement. it allowed information to be input not at once, but anyhow and showed results immediately. This explains why it is being used till date.

The CLI is programmable, efficient in its use of network bandwidth nd you can use them to edit files in a remote system without having to first copy them into your ocal machine. In general you can be much more expressive with the CLI than you could ever be with pointing and clicking.

Command Line Interface

There are command line systems everywhere. Google Chrome has a developer tool section that provides a CLI. Python also provides a CLI to use with their inteprator ( Python's interactive interpreter.). A Command Line Interface therefore is any computer interface where the user enters textual commands and gets textual responses back.

Whilst CLIs vary significantly, proficiency in one will give you a head start learning another.

In this tutrial, we mostly wore with the Linux Command Line Interface in a VM, however, in order to access that you need to use your OS’s host OS’s command inteface.

Page 4: Terminal vs The Shell

One thing to underatnd isn the dfferent between the terminal and the shell.

The terminal is a program that draws text in its windows and lets you type things in with the keyboard.

The termnal itself doesnt know what to do with that input. It relies on the shell for that. WHen you tye things into the terminal, it sends what is typed to The Shell which is a separate program.

When you press Enter, the shell interprets what you typed in as a command, inteprates that command, figures out what program you want to run, runs it and sends the output nack to the terminal so you can view it.

Although there are a lot of shell programs you can choose from such as TCSH, KSH and Seashell, etc. they all. work very slightly and have various different features

The default one on the Linux system and the Mac is called the GNU bash. For this tutorial we will be using the GNU Bash.

NOTE: You can also use the shell wthout the terminal. You can write shell commands into a file and arrange for your to computer to run the shell program on that file. This is called shell scripting and that is beyoud the scope of this programmer. LOL.
